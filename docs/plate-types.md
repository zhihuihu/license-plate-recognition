# 车牌类型与识别边界

本文说明当前项目“代码格式校验覆盖什么”和“模型实际可能识别什么”的区别。

车牌识别效果不仅取决于牌型，还取决于图片中车牌的像素大小、清晰度、角度、光照、反光、遮挡和车牌是否被其他车辆或文字干扰。下表是当前工程能力说明，不是对所有图片 100% 正确的承诺。正式上线前必须使用目标停车场的真实图片验收。

## 当前支持矩阵

| 车牌类型 | 当前处理方式 | 状态 | 备注 |
| --- | --- | --- | --- |
| 普通蓝牌小型汽车 | HyperLPR3 主识别，低置信度进入复核链 | 较适合 | 当前最主要的常规场景。需要车牌完整、清晰、无遮挡。 |
| 普通黄牌大型汽车 | 主模型识别和统一格式校验 | 较适合 | 远距离、强反光、倾斜严重时需要现场验证。 |
| 新能源小型车牌 | 支持 8 位新能源车牌格式 | 较适合 | 绿色车牌颜色本身不参与最终格式判断，图片质量仍然决定结果。 |
| 新能源大型车牌 | 支持常见 8 位格式 | 需验证 | 不同相机、距离和牌面布局需要用现场图片测试。 |
| 普通摩托车牌 | 复用普通省份车牌格式校验 | 需验证 | 受车牌尺寸、倾斜角度和车架遮挡影响较大。 |
| 摩托车双层黄牌 | 尝试合并上行“省份+字母”和下行数字/字母 | 需验证 | 上下行必须都被检测到；错位、遮挡或下行粘连时可能失败。 |
| 学练车牌 | 支持末尾 `学` 的常见格式 | 需验证 | OCR 必须正确识别专用汉字，不能只看格式通过。 |
| 警用车牌 | 支持末尾 `警` 的部分常见格式 | 需验证 | 警用牌型存在特殊号段，当前不是完整号段规则库。 |
| 挂车牌 | 支持末尾 `挂` 的常见格式 | 需验证 | 需要目标车辆和相机角度测试。 |
| 试验车牌 | 支持末尾 `试` 的常见格式 | 需验证 | 临时/纸质标识不等同于标准车牌，当前不保证。 |
| 港澳入出境车牌 | 支持类似 `粤Z1234港`、`粤Z1234澳` 的格式 | 需验证 | 仅覆盖常见入出境格式，不代表所有港澳车辆号牌。 |
| 武警 `WJ` 车牌 | 支持类似 `WJN22628` 的清洗和校验 | 需验证 | 具体号段、字体和特殊版式不做完整规则承诺。 |
| 使馆/领馆车牌 | 支持常见 `使...`、`领...` 前缀格式 | 需验证 | 只覆盖常见 OCR 文本形态，不是完整外交车辆规则库。 |
| 部分军队及军用车辆牌 | 支持部分 `军/海/空/北/南/沈/兰/济/广/成` 前缀形态 | 需验证 | 当前只做宽松 OCR 层校验，不包含完整军牌编制号段。 |

状态含义：

- **较适合**：属于主模型和常规格式的主要目标，但仍需要现场数据验收。
- **需验证**：代码可以尝试清洗或组合结果，但模型召回率受牌型和图片条件影响明显。
- **不保证**：通过格式校验只说明字符串形状合理，不代表图片识别一定正确。

## 当前不保证或不覆盖的情况

以下场景不能作为当前服务的稳定能力对外承诺：

- 严重模糊、过曝、欠曝、夜间无补光、强反光或车牌像素过小
- 车牌被泥土、车架、拖车钩、广告贴纸或其他物体遮挡
- 极端侧拍、透视变形、车牌弯曲或上下行分离严重
- 图片中存在多辆车，但没有额外的车辆/车牌定位业务规则
- 港澳本地号牌、台湾号牌、外国号牌及不在当前规则中的其他地区号牌
- 纸质临时牌、临时通行证、无法形成标准车牌字符序列的标识
- 完整外交、军用、警用等特殊号段规则校验
- 仅凭颜色判断蓝牌、黄牌、绿牌；颜色不能替代 OCR 和格式校验

## 结果可信度建议

`confidence` 是当前 OCR 候选的模型置信度，不是业务准确率。建议业务侧：

1. 先用真实现场图片建立按牌型、天气、时段和相机的测试集。
2. 分别统计“检测到车牌”“字符全部正确”“整张车牌完全正确”三个指标。
3. 对双层摩托车牌、夜间、逆光和特殊车牌单独设定验收阈值。
4. 低置信度或特殊牌型进入人工复核，不要直接用于收费、放行或处罚决策。
5. 需要多辆车、视频流和进出场判断时，在服务外增加多帧投票和车辆事件编排。
